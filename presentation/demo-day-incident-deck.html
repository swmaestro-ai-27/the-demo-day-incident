CASE 01

The Demo Day Incident

AI가 범인을 맞히는 게임이 아니라, AI들의 추리를 의심하는 게임.

NPC가 단서를 주는 사람이 아니라,
자기 나름대로 범인을 찍는 사람이라면?
01 / 07

Core Idea

그럴듯하게 말하지만, 전부 믿을 수는 없습니다

같은 사건을 보고도 백엔드, 프론트엔드, AI Agent 담당자는 서로 다른 근거로 다른 사람을 의심합니다.

백엔드 담당자

서버 기록과 권한 실행 흐름을 중심으로 사건을 봅니다.

프론트엔드 담당자

발표 자료 삭제와 화면 흐름을 중심으로 의심합니다.

AI Agent 담당자

Agent 실행 기록과 Tool 호출 맥락을 중심으로 판단합니다.

02 / 07

Difference

기존 추리 게임과 다른 지점

기존 NPC

단서 전달자

정해진 대사를 말하고, 플레이어는 그 정보를 수집합니다.

우리 NPC

불완전한 추리자

자기 정보와 관점 안에서 추리합니다. 그래서 맞을 수도, 틀릴 수도 있습니다.

재미는 “정답 묻기”가 아니라 “진짜 근거 고르기”입니다.

03 / 07

Play Loop

플레이어는 AI를 의심합니다

1. 단서 확인 잠금 기록, 삭제 기록, 실행 기록처럼 사건의 조각을 엽니다.
2. 인물 대화 열린 단서를 바탕으로 관련 인물 Agent에게 질문합니다.
3. 추리 비교 누가 무엇을 알고, 무엇을 모르는지 비교합니다.
4. 최종 제출 범인, 동기, 결정적 증거를 직접 선택합니다.

사건 접속 → 단서 확인 → 인물 질문으로 이어지는 짧은 조사 흐름.

다듬는 중 단서/인물 데이터, 사건 설명 이벤트, 추리 성공/실패 이벤트.
